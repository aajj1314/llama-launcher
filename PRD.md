## 1. 背景

### 1.1 项目现状

llama-launcher 是一个基于 Python 的 llama.cpp 模型启动管理工具，当前版本 (v3.1) 具有以下特性：

- **赛博朋克风格 TUI 界面**：使用 ANSI 转义序列实现霓虹配色和字符边框
- **三种运行模式**：CLI 交互模式、Server API 模式、Embedding 模式
- **实时指标监控**：显示 Token 处理速度、上下文使用情况
- **动态模型扫描**：自动检测 models 目录下的 GGUF 模型文件
- **WebUI 支持**：基于浏览器的远程管理界面
- **模块化架构**：清晰、组织良好的代码结构
- **配置管理**：外部配置文件支持
- **增强错误处理**：健壮的错误管理

### 1.2 已解决问题

1. **单一终端依赖**：已添加 WebUI 界面，支持浏览器访问
2. **缺乏 Web 访问能力**：已实现基于 FastAPI 的 Web 服务
3. **状态同步缺失**：已实现基于单例模式的共享状态管理器
4. **配置灵活性不足**：已实现外部配置文件和环境变量支持

### 1.3 业务需求

用户提出以下优化需求：

1. 构建 WebUI 界面（监听地址：`0.0.0.0:8087`）- 已完成
2. WebUI 与 TUI 需要同时同步运行 - 已完成
3. 优化现有代码逻辑 - 已完成
4. 由于本地未部署 llama.cpp，优化代码时需慎重 - 已考虑

## 2. 目标

### 2.1 总体目标

将 llama-launcher 从单一的 TUI 应用扩展为 TUI + WebUI 双界面应用，实现：

- **双界面并行**：TUI 和 WebUI 同时运行，状态实时同步
- **远程管理能力**：通过 Web 浏览器远程启动、停止、监控模型
- **代码质量提升**：优化架构和逻辑，提高可维护性和稳定性

### 3.2 功能详细描述

#### F1.1 Web 服务

- 使用 FastAPI 框架搭建轻量级 Web 服务
- 监听 `0.0.0.0:8087`，支持局域网访问
- 提供 RESTful API 供前端调用
- 静态文件服务承载前端页面

#### F1.2 模型列表展示

- 自动扫描 `models` 目录下的 `.gguf` 文件
- 显示模型名称、文件大小
- 支持选中模型进行启动操作

#### F1.3 模型启动/停止

- 启动：调用 `run_cli()` / `run_server()` / `run_embedding()` 启动模型
- 停止：调用 `terminate_process()` 终止进程
- 启动参数从共享配置中读取

#### F1.4 实时监控面板

- 通过轮询获取服务器日志解析结果
- 显示指标：PROMPT TOK、EVAL TOK、PPD、EPD、CONTEXT 使用率
- 定时刷新（1 秒间隔）

#### F1.5 配置参数调整

- Context 大小选项：4k, 8k, 16k, 32k, 64k, 128k
- NGL 选项：0, 33, 66, 99, 999
- 端口配置：1-65535
- 运行模式：CLI、Server、Embedding

#### F2.1 共享状态管理器

- 使用 Python 单例模式实现状态共享
- 状态包括：
  - 当前运行的模型信息（进程 PID、模型名称、运行模式）
  - 当前配置（Context、NGL、Port、Mode）
  - 服务器指标（从日志解析）
  - 模型列表

#### F4.4 模块化架构重构

- 配置管理模块 (`config.py`)：管理应用配置
- 状态管理模块 (`state_manager.py`)：管理应用状态
- 进程管理模块 (`process_manager.py`)：管理模型进程
- 工具模块 (`utils.py`)：提供通用工具函数
- TUI 模块 (`run.py`)：提供终端界面
- WebUI 模块 (`web_app.py`)：提供 Web 界面
- 启动模块 (`launcher.py`)：统一启动脚本
