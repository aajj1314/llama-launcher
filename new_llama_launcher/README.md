# Llama Launcher v4.0 - 用户友好版

一个简单易用、设计人性化的 llama.cpp 模型启动器。

## 主要特性

✅ **不会自动重置用户输入** - 修改的参数会保留，直到手动保存
✅ **配置持久化** - 所有配置保存到 `~/.llama_launcher_v4/config.json`
✅ **进程日志管理** - 日志写入文件，不会刷屏终端
✅ **清晰的UI设计** - 渐变色彩，响应式布局
✅ **完整的API接口** - FastAPI 提供RESTful服务

## 项目结构


## 安装使用

### 1. 安装依赖

```bash
cd new_llama_launcher
pip install -r requirements.txt
```

### 2. 启动服务

```bash
python main.py
```

或者指定端口：

```bash
python main.py --host 0.0.0.0 --port 8000
```

### 3. 访问界面

在浏览器中打开：`http://localhost:8000`

## 使用说明

### 第一次使用

1. 打开网页
2. 在"路径配置"区域设置你的 llama.cpp 目录路径
3. 点击"更新路径"
4. 系统会自动扫描 models 目录下的 GGUF 模型
5. 选择一个模型
6. 配置运行参数
7. 点击"保存配置"
8. 点击"启动"

### 用户界面特点

- **不会自动重置** - 你在输入框中修改的值会保留，直到点击"保存配置"
- **明确的操作** - 所有修改都需要明确的点击确认
- **即时反馈** - 状态变化会实时显示
- **日志记录** - 进程输出写入文件，不会刷屏

## 配置项

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| llama_cpp_path | llama.cpp 目录路径 | ~/llama.cpp |
| run_mode | 运行模式 (cli/server/embedding) | cli |
| context_size | 上下文大小 | 4096 |
| ngl | GPU 层数 | 0 |
| port | Server 模式端口 | 8000 |
| web_host | Web服务地址 | 0.0.0.0 |
| web_port | Web服务端口 | 8000 |

## 技术设计

### 前端优化

- 使用本地状态缓存，避免自动重置用户输入
- 智能更新逻辑：只在值真正变化时才更新输入框
- 响应式设计，支持手机和桌面

### 后端架构

- 模块化设计：配置、模型、状态、Web 分离
- 单例模式管理全局状态
- 线程安全的状态访问
- 完整的错误处理

### 进程管理

- 日志重定向到文件，避免终端刷屏
- 进程独立运行，不影响主服务
- 优雅的启动/停止/重启流程

## 常见问题

**Q: 我的模型在哪里？**
A: 确保你的 GGUF 模型放在 llama.cpp 目录下的 models 文件夹中。

**Q: 参数修改后为什么没有生效？**
A: 记得点击"保存配置"按钮！我们不会自动保存你的修改。

**Q: 启动后终端没有输出？**
A: 对！这是设计特性，日志会写入文件，在 Web 界面的"运行日志"区域查看。

**Q: 如何停止服务？**
A: 在 Web 界面点击"停止"按钮，或者在终端按 Ctrl+C。
