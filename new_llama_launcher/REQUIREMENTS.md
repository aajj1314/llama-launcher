# Llama Launcher 功能需求文档

## 1. 项目概述
- **项目名称**：Llama Launcher (重新设计版)
- **版本**：v4.0
- **目标**：提供一个用户友好、功能完整的 llama.cpp 模型启动器
- **定位**：零配置、即开即用、用户体验优先

## 2. 核心功能需求

### 2.1 模型管理模块
- **需求 2.1.1**：自动扫描 models 目录下的所有 GGUF 格式模型
- **需求 2.1.2**：显示模型详细信息（名称、大小、路径）
- **需求 2.1.3**：支持手动刷新模型列表
- **需求 2.1.4**：模型列表按名称排序显示

### 2.2 配置管理模块
- **需求 2.2.1**：运行模式选择
  - CLI 模式（交互式）
  - Server 模式（API 服务）
  - Embedding 模式（向量生成）
- **需求 2.2.2**：上下文大小配置
  - 预定义选项：4k, 8k, 16k, 32k, 64k, 128k
- **需求 2.2.3**：GPU 层配置（NGL）
  - 预定义选项：0, 33, 66, 99, 999
- **需求 2.2.4**：端口配置（Server 模式）
  - 默认：8000
  - 允许自定义修改
- **需求 2.2.5**：日志级别配置
  - Debug, Info, Warning, Error
- **需求 2.2.6**：超时配置（秒）
  - 默认：30 秒
- **需求 2.2.7**：GPU 内存限制配置（MB）
- **需求 2.2.8**：配置持久化保存和加载
- **需求 2.2.9**：配置变更立即生效，无需重启

### 2.3 路径配置模块
- **需求 2.3.1**：llama.cpp 路径设置
- **需求 2.3.2**：路径验证（检查目录是否存在）
- **需求 2.3.3**：路径变更后自动重新扫描模型
- **需求 2.3.4**：显示当前配置的路径

### 2.4 进程管理模块
- **需求 2.4.1**：启动模型进程
  - 显示启动状态
  - 验证二进制文件存在
- **需求 2.4.2**：停止模型进程
  - 正常停止流程
  - 超时强制停止
- **需求 2.4.3**：重启模型进程
- **需求 2.4.4**：进程状态监控
  - 运行状态
  - PID
  - 启动时间
- **需求 2.4.5**：日志显示和管理
  - 实时更新
  - 滚动显示
  - 日志旋转（防止文件过大）
- **需求 2.4.6**：输出控制（防止终端刷屏）

### 2.5 WebUI 模块
- **需求 2.5.1**：响应式用户界面
  - 支持桌面和移动端
- **需求 2.5.2**：表单输入验证
  - 实时验证
  - 友好错误提示
- **需求 2.5.3**：配置变更立即生效
  - 表单输入后立即更新
  - 保存按钮确认
- **需求 2.5.4**：实时状态更新
  - WebSocket 或轮询
  - 状态平滑过渡
- **需求 2.5.5**：用户友好的错误提示
  - 明确的错误信息
  - 解决方案建议
- **需求 2.5.6**：禁用自动复位功能
  - 用户输入保留
  - 数值修改正常工作

### 2.6 CLI/TUI 模块
- **需求 2.6.1**：终端界面友好
  - 颜色编码
  - 清晰布局
- **需求 2.6.2**：键盘快捷键支持
  - 方向键导航
  - 回车确认
  - 其他常用操作
- **需求 2.6.3**：输出控制
  - 防止刷屏
  - 分页显示
  - 可配置的输出级别
- **需求 2.6.4**：状态更新清晰可见

### 2.7 统一启动模块
- **需求 2.7.1**：支持单独启动 TUI
- **需求 2.7.2**：支持单独启动 WebUI
- **需求 2.7.3**：支持同时启动 TUI 和 WebUI
- **需求 2.7.4**：命令行参数解析
  - --mode [tui/web/both]
  - --port [端口号]
  - --host [主机地址]
- **需求 2.7.5**：优雅关闭流程

## 3. 技术需求

### 3.1 后端技术栈
- **Python 3.8+**
- **FastAPI**（Web 框架）
- **Uvicorn**（ASGI 服务器）
- **标准库**（尽量使用，减少依赖）

### 3.2 前端技术栈
- **原生 HTML/CSS/JavaScript**（降低依赖复杂性）
- **或者**：轻量级框架（Vue 3 / React）

### 3.3 代码质量
- **模块化设计**
- **类型提示**
- **错误处理完善**
- **日志记录完整**

### 3.4 部署友好
- **单文件启动器**
- **清晰的依赖说明**
- **简洁的 README**

## 4. 非功能性需求

### 4.1 性能
- **启动时间**：< 2 秒
- **内存占用**：< 100MB（空闲状态）
- **响应延迟**：< 100ms

### 4.2 稳定性
- **崩溃恢复**
- **异常处理完善**
- **无内存泄漏**

### 4.3 可维护性
- **代码结构清晰**
- **注释充分**
- **遵循最佳实践**

### 4.4 用户体验
- **直观的操作流程**
- **友好的错误提示**
- **响应式设计**

## 5. 优先级
- **P0（核心功能）**：必须实现
  - 模型扫描和选择
  - 基本配置管理
  - 进程启动和停止
  - 配置保存和加载
- **P1（重要功能）**：应该实现
  - 路径配置
  - WebUI
  - TUI
  - 日志显示
- **P2（优化功能）**：可以实现
  - 高级配置选项
  - 更多模型信息显示
  - 统计功能
